# Инструкция по начальной настройке Raspberry Pi 2b

## 1. Выбор дистрибутива
Самый последний нормальный дистрибутив с GUI для Raspberry Pi 2b (raspi2b) это bullseye. Далее все дитрибутивы идут с очень тормозным GUI.

[Скачать bullseye можно тут](https://downloads.raspberrypi.com/raspios_oldstable_armhf/images/raspios_oldstable_armhf-2025-05-07/)

## 2. Установка
* Скачиваем и устанавливаем [Raspberry Pi Imager](https://www.raspberrypi.com/software/)
* Запускаем и выбираем Raspberry Pi 2
* На выборе операционной системы надо выбрать в самом низу "Использовать настраиваемый образ"
* Выбираем скаченый ранее образ bullseye
* Выбираем sd-карту на которую надо установить.
* Устанавливаем.

## 3. Вероятная проблема
У меня была проблема с синхронизацией времени. Из-за этого в GUI не давало обновить пакеты.
* Я отключил проверку времени для обновлений в GUI:
	```
	sudo apt purge pi-package -y
	```
* Включил другую синхронизацию времени, с помощью альтернативного NTP клиента
* Установка клиента
	```
	sudo apt install ntp -y
	```
* Запуск:
	```
	sudo systemctl enable ntp --now
	```
* И проверка, если надо:
	```
	ntpq -p
	```

## 4. Моя проблема - драйвера WiFi
У меня какой-то китайский донгл с алиэкспресса. В каких-то дистрибутивах драйвера на него сразу подхватываются, в каких-то нет. Но как оказалось есть ещё одна проблема, даже если в GUI всё подхватилось и работает, то в консоли нет. Решением является установка правильного драйвера.

Далее описана установка драйвера для моего донгла 8188eu.
Т.к. у меня драйвера в GUI подхватились, то именно из GUI все следующие команды сделал. Иначе бы пришлось искать провод LAN.

* Установка заголовков ядра:
	```
	sudo apt install -y raspberrypi-kernel-headers git bc build-essential dkms
	```
* Скачиваем драйвер:
	```
	git clone https://github.com/aircrack-ng/rtl8188eus
	```
* Переходим в каталог:
	```
	cd rtl8188eus
	```
* Устанавливаем драйвер через DKMS:
	```
	sudo make dkms_install
	```
* Перезагрузка:
	```
	sudo reboot
	```
* После перезагрузки для проверки:
	```
	ip a
	```
* Должно быть что-то такое:
	```
	wlan0: ... state UP ... inet 192.168.x.x
	```
* Теперь будет работать стабильно, как в GUI, так и в консоли.

## 5. Помогаем SD-карте не умереть

На Raspberry Pi 2 SD карты умирают не от возраста, а от постоянных мелких записей, которые делает система. Карту нужно покупать как минимум с пометкой High Endurance (позицианируются для видеорегистраторов), главное не нарваться на подделку.

### 5.1. Включить zram (уменьшает запись на карту)
ZRAM хранит swap в RAM, сжатый. Это главная защита SD-карты.
* Установить:
	```
	sudo apt install zram-tools -y
	```
* Проверить:
	```
	cat /proc/swaps
	```
* Должно быть что-то вроде:
	```
	/dev/zram0
	```

### 5.2. Отключить swap-файл на SD-карте
Если zram включён, то swapfile больше не нужен.
```
sudo dphys-swapfile swapoff	
sudo dphys-swapfile uninstall	
sudo systemctl disable dphys-swapfile	
```

### 5.3. Перевести /var/log в RAM (log2ram)
Логи постоянно пишутся и убивают карту.
* Установить:
	```
	wget https://github.com/azlux/log2ram/archive/master.zip
	unzip master.zip
	cd log2ram-master
	sudo ./install.sh
	```
* Проверить:
	```
	mount | grep log2ram
	```
* Должно быть что-то вроде: 
	```
	log2ram on /var/log type tmpfs
	```

### 5.4. Отключить journald persistent logs
По умолчанию systemd пишет огромные логи.
* Отключить:
	```
	sudo nano /etc/systemd/journald.conf
	```
* Найти строку:
	```
	#Storage=auto
	```
* Заменить на:
	```
	Storage=volatile
	```
* Сохранить (ctrl+o, ctrl+x)
* Перезагрузить
* Теперь systemd пишет логи только в RAM.

### 5.5. Отключить ненужные сервисы, которые пишут на диск
Тут смотреть, если чтото надо, то не отключать
```
sudo systemctl disable --now triggerhappy
sudo systemctl disable --now bluetooth
sudo systemctl disable --now avahi-daemon
sudo systemctl disable --now cups
```

### 5.6. Уменьшить частоту записи времени (fsync)
* Включить оптимизацию ext4:
	```
	sudo tune2fs -o journal_data_writeback /dev/mmcblk0p2
	```
* Открыть файл
	```
	sudo nano /etc/fstab
	```
* Добавить к корневому разделу (тот который просто косая черта /):
	```
	noatime,nodiratime
	```
* Сохранить (ctrl+o, ctrl+x)
* Это уменьшает количество записей при чтении файлов.

### 5.7. Включить кэширование APT (меньше перезаписей)
* Открыть файл
	```
	sudo nano /etc/apt/apt.conf.d/02nocache
	```
* Вставить:
	```
	Dir::Cache "";
	Dir::Cache::archives "";
	```
* Сохранить (ctrl+o, ctrl+x)

### 5.8. Не хранить историю bash на диске
Сомнительная вещь, но имеет место быть.
* Выполнить команду
	```
	echo 'HISTFILE=/dev/null' >> ~/.bashrc
	```
По итогу история хранится только в текущем bash окне. После перезагрузки (или просто в другом bash окне) история будет отсутствовать.

## 6. Удаленный доступ по SSH
* Можно включить тумблер SSH в настройках в GUI.
* Вариант через консоль
	```
	sudo systemctl enable ssh
	sudo systemctl start ssh
	```
* Проверить статус
	```
	systemctl status ssh
	```
* Коннектиться как-то так
	```
	ssh mylogin@192.168.1.11
	```
## 7. Удаленный стол RDP (GUI)
Т.к. я не планирую подключать raspi2b к монитору, а коннектиться удаленно, то мне нужно сделать так чтобы при старте загружалась консоль, а не GUI.

* Заходим в конфиги
	```
	sudo raspi-config
	```
* Далее 
	```
	System Options → Boot / Auto Login → выбрать консоль с автовходом (или без)
	```
* Перезапуск

Чтобы был GUI при удаленном подключении нужен RDP

* Ставим XRDP
	```
	sudo apt update
	sudo apt install xrdp -y
	```
* После установки XRDP проверить, что GUI не запускается автоматически
	```
	sudo systemctl set-default multi-user.target
	```

При подключении по XRDP у меня были какието приколы с тем что выбран язык не английский, а логин и пароль на английском и я ввести ничего не могу и язык не переключается. Поэтому надо ещё задать язык английский принудительно. Хотя по ощущениям проблема была ещё и в том, что окно подключения (в windows) было запущено не с английским. То есть возможно надо включить английский, а потом уже запускать "Подключение к удаленному рабочему столу".

* Выполнить:
	```
	sudo localectl set-x11-keymap us
	sudo systemctl restart xrdp
	```
Чтобы не плодить GUI сеансы, надо сделать так чтобы был только один сеанс с GUI.

* Установить xorgxrdp
	```
	sudo apt install xorgxrdp -y
	```
* Включить “single session mode” в конфиге XRDP:
	```
	sudo nano /etc/xrdp/xrdp.ini
	```
* Найти (если нету, то добавить в секцию Globals):
	```
	max_sessions=
	```
* Поменять на:
	```
	max_sessions=1
	```
* Это заставляет XRDP использовать один сеанс на пользователя.
* Включить reconnect. В том же конфиге (если нету, то добавить в секцию Globals):
	```
	policy=
	```
* Поменять на:
	```
	policy=allow_reconnect
	```
* Сохранить (ctrl+o, ctrl+x)
* Перезапустить XRDP
	```
	sudo systemctl restart xrdp
	```
Для подключения можно использовать встроенное в Windows "Подключение к удаленному рабочему столу"
* Адрес: raspberrypi (или айдишник).
* Можно указать свой логин: mylogin.
* Пароль лучше вводить в окне логина на raspi.
